# video-studio 热梗库 (reference-memes.md)

> 跟 `reference-style-video.md` 平级. LLM 每次跑 job 由 prompt 引导 read-and-use.
> 调试阶段: 库里 5 条人工筛选的精品段子, LLM **直接抄原文嵌入**, 不仿写不自创.

## 入选门槛 (4 条, 必须全过)

1. **钩子必须是该人物广为人知的真实特征** (残疾 / 死亡 / 失败 / 骂名), 不能是抽象类比
2. **必须双关或同音字 PUN** (一个意思挂历史, 一个意思挂现代), 改了" 段子" 也成立的 = 失败
3. **恶趣味 > 谐音 > 知识梗** (越黑越上头)
4. **不能只是"古人+现代活动" 的空贴**

## 段子表 (5 条, 锁定, 全部用 verbatim, 不改字)

| # | 段子 | 种子 | PUN 词 | 场景 | 格式 |
|---|---|---|---|---|---|
| 1 | 魏忠贤谈 KPI——无后为大 | 魏忠贤 (太监) | 无后 | KPI 汇报 | 声明式 |
| 2 | 夏侯惇相亲——一眼就相中 | 夏侯惇 (眼瞎) | 一眼 | 相亲 | 声明式 |
| 3 | 路易十六很讲义气——我有事了他一定替我出头 | 路易十六 (断头) | 出头 | 朋友 | 声明式 |
| 4 | 路易十六的困惑——摸不着头脑 | 路易十六 (断头) | 头脑 | 日常 | 声明式 |
| 5 | 路易十六为什么不能上高速? 因为高速禁止调头 | 路易十六 (断头) | 调/掉 同音 | 高速 | 问/答 |

**使用规则** (重要):
- LLM **直接抄** 段子原文嵌入脚本, **不改字不重写**
- LLM **自己判断** 哪个段子适合当前脚本主题 (例如脚本讲打工人/KPI 才用 #1, 讲相亲才用 #2, 讲交通/出行才用 #5)
- **不合适就完全不用**, 5 条全部不合适 = 0 段子, 也 OK
- **不引入新段子** (LLM 严禁自创, 用户后续手动补库)

## 调性边界

- **准**: 反派 (盖章坏人, 如魏忠贤) + 戏说古人 (路易十六 / 夏侯惇 - 非正史英雄)
- **不准**: 悲剧英雄 (项羽 / 岳飞 / 关羽) + 革命先烈 + 受害者
- **不准**: 治愈 / 松弛 / 愿你 / 希望 (style doc 已有, 引用即可)

## 维护纪律

- **不要堆数量**: 5-10 条精品就够, 50 条水货稀释 LLM 注意力
- **看到弱段子就删**: 直接编辑 markdown 表格删, 不留次品
- **种子是稀有资源**: 加新人物 = 严重决定, 必须能举出 1 个" 同音字 PUN" + 1 个" 现代场景" 才加
- **格式不只声明式**: 看到好 Q&A 立刻加, 增加多样性

## 怎么补梗 (4 步, 零重启)

1. 打开 `reference-memes.md` — vim / edit 任意方式
2. 加一行到" 段子表" 表格 (按 6 列格式: # / 段子 / 种子 / PUN 词 / 场景 / 格式)
3. git commit (跟其他 reference 文档一起走, 不需要碰代码)
4. 下一条 job 自动生效 — LLM 每次跑都从硬盘重新 read
